File Review System Optimisation

Streamlining the compliance workflow for 160+ file reviewers to handle high-density question sets with precision.

The Scale of the Problem

The system had to support 160 advisors organised into teams of 18 people. The existing process was buckling under the weight of massive datasets, where a single file review could contain up to 83 specific compliance questions. We started by mapping the flow to identify bottlenecks.

Visualising the Bottlenecks

To understand why the system was buckling, we first mapped the existing workflow (shown below). The diagram reveals a rigid, linear pipeline trying to support 160 advisors and 18-person review teams. We discovered that a single file could trigger up to 83 unique compliance questions, creating massive "traffic jams" at every handoff point between the swimlanes.

Managing Cognitive Load

With over 1,000+ potential case combinations, reviewers were suffering from "scroll fatigue." I introduced a fixed Sidebar Navigation that split the "Three Main Screens" into manageable chunks. This allowed users to jump between the Fact Find, Remedial Actions, and Outcome sections without losing context.

Team Workflows

For the 18-person teams, visibility was key. The dashboard needed to clearly separate "Allocated" reviews from the "To-Do" backlog. We used colour-coded status pills to allow Team Leads to spot 'At Risk' files instantly.

Defining the Outcome

The final step of the flow is the "Outcome" screen. We redesigned this to be binary and clear. Instead of ambiguous text fields, we implemented structured data inputs for Remedial Actions, ensuring that any failed review had a specific, trackable reason attached to it.

The Challenge

Revolution is a document management platform serving 160+ advisors. As AI capabilities became more prevalent, we faced a critical challenge: how to leverage AI for improved efficiency while maintaining compliance standards and user trust. The existing system couldn't handle the nuanced requirements of AI-generated content validation.

Design Process

I led a comprehensive design process that involved:

  • 1 Research & Discovery: Conducted 20+ user interviews with advisors and compliance officers to understand pain points and requirements.
  • 2 System Architecture: Designed a 'Static Snapshot' system that captures AI-generated content at a specific point in time for compliance verification.
  • 3 UI/UX Design: Created intuitive interfaces for content review, approval workflows, and compliance reporting.
  • 4 Testing & Iteration: Conducted extensive user testing and iterated on the design based on feedback.

Key Outcomes

✓ 40% Reduction in Review Time

By automating routine compliance checks and creating clearer workflows, advisors spend 40% less time on document review.

✓ 99.8% Compliance Rate

The system maintains near-perfect compliance through automated validation and expert review workflows.

✓ Improved User Trust

Users report significantly higher confidence in AI-assisted workflows due to transparent compliance systems.

Lessons Learned

This project reinforced several key principles for designing AI systems:

  • → Transparency is critical: Users need to understand how AI systems make decisions and how compliance is maintained.
  • → Design for edge cases: The 0.2% of edge cases often reveal the most important design insights.
  • → Involve stakeholders early: Compliance officers, legal teams, and end users all have valuable perspectives.
